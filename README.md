# TerrainFormer

Autonomous off-road navigation using World Models and Decision Transformers.

## Architecture

```
LiDAR Point Cloud
       │
       ▼
BEV Projection (256×256×64)          ← PointPillars ~5 ms
       │
       ▼
World Model
  ├── TerrainTokenizer  → patch embeddings
  ├── DynamicsTransformer → latent state
  └── PredictionHeads
        ├── TraversabilityHead  (256×256, sigmoid)
        ├── ElevationHead       (256×256)
        ├── SemanticHead        (256×256, C classes)
        └── FuturePredictionHead (10 future frames)
       │
       ▼
Decision Transformer
  ├── 10 past frames  (from rolling buffer)
  ├── 1 current frame
  └── 10 future frames (predicted above)
       │
       ▼
Action (0–11)
```

## Training Phases

| Phase | Script flag | Datasets | What trains | Output |
|-------|------------|----------|-------------|--------|
| 1 — World Model | `pretrain_world_model.yaml` | LidarDustX + GOOSE-3D | Traversability, elevation, semantics, future prediction | `outputs/world_model_pretrain/` |
| 2 — Decision | `train_decision.yaml` | RELLIS-3D (seqs 0–4) | Action prediction (world model frozen) | `outputs/decision_train/` |
| 3 — Joint Fine-tune | `finetune_joint.yaml` | RELLIS-3D | End-to-end (optional) | `outputs/finetune_joint/` |

## Action Space (12 actions)

| ID | Name | Description |
|----|------|-------------|
| 0 | Stop | Stationary (v < 0.1 m/s) |
| 1 | Fwd Slow | Forward, slow (0.1–0.5 m/s), minimal turn |
| 2 | Fwd Med | Forward, medium (0.5–1.0 m/s), minimal turn |
| 3 | Fwd Fast | Forward, fast (≥ 1.0 m/s), minimal turn |
| 4 | Left Sharp | Turn left, sharp (ω ≥ 0.6 rad/s) |
| 5 | Left Med | Turn left, medium (0.3–0.6 rad/s) |
| 6 | Left Slight | Turn left, slight (0.1–0.3 rad/s) |
| 7 | Right Slight | Turn right, slight |
| 8 | Right Med | Turn right, medium |
| 9 | Right Sharp | Turn right, sharp |
| 10 | Fwd+Left | Fast forward + sharp left (combined) |
| 11 | Fwd+Right | Fast forward + sharp right (combined) |

Classification logic: turn rate checked first; if |ω| ≥ 0.1 rad/s and v ≥ 1.0 m/s → Fwd+Turn; if |ω| ≥ 0.1 rad/s → Turn; else → Forward speed bucket.

## Dataset Setup

### RELLIS-3D

Dataset location: `/home/rickslab3/Documents/Datasets/RELLIS/`

```
RELLIS/
└── dataset/
    └── sequences/
        ├── 00000/
        │   ├── velodyne/          # .bin point clouds (2842 frames)
        │   │   ├── 000000.bin
        │   │   └── ...
        │   ├── labels/            # Semantic label files (.label)
        │   ├── poses.txt          # 3×4 [R|t] per line
        │   └── calib.txt
        ├── 00001/ ... 00004/      # Same structure
        └── actions/               # Generated by generate_action_labels.py
            └── 00000/
                ├── actions.npy          # (N,) int64
                ├── goals.npy            # (N, 2) float32 [dx, dy]
                └── actions_metadata.json
```

Pose file format: one line per frame, 12 space-separated floats = 3×4 transformation matrix `[R | t]`.
Extract position as `pose[:, 3]` and yaw as `arctan2(pose[1,0], pose[0,0])`.

## Scripts Reference

### `scripts/generate_action_labels.py` — Auto-label from poses

```bash
python scripts/generate_action_labels.py
```

### `scripts/label_actions.py` — Interactive Action Labeler UI

Manual annotation tool. Load a folder of `.bin` files and a `poses.txt`, then assign actions frame-by-frame using buttons or keyboard.

```bash
# Interactive (use the Load buttons inside the UI)
python scripts/label_actions.py

# Pre-load data from command line
python scripts/label_actions.py \
    --folder /data/RELLIS/bin/00000 \
    --poses  /data/RELLIS/calib/00000/poses.txt
```

**UI layout:**

```
┌──────────────────────────┬────────────┬──────────────────────┐
│                          │ Frame Info │  ■  Stop             │
│   BEV Point Cloud        │ Frames 1–20│  ↑  Fwd Slow         │
│   (white background,     │ X: +12.34m │  ↑  Fwd Med          │
│    viridis height color) │ Y:  +4.56m │  ⇑  Fwd Fast         │
│                          │ Z:  +0.12m │  ↺  Left Sharp       │
│  [labeled badge top-left]│ Yaw: +23°  │  ←  Left Med         │
│  [frame counter top-right│ ...        │  ↖  Left Slight      │
│                          ├────────────┤  ↗  Right Slight     │
│       ▲ EGO (red)        │ Trajectory │  →  Right Med        │
│                          │ (mini path │  ↻  Right Sharp      │
│                          │  + yellow  │  ↰  Fwd+Left         │
│                          │  batch     │  ↱  Fwd+Right        │
│                          │  segment)  │                      │
│                          ├────────────┤                      │
│                          │ Progress   │                      │
│                          │ ████ 28%   │                      │
│                          │ action dist│                      │
├──────────────────────────┴────────────┴──────────────────────┤
│[Load Folder][Load Poses][←Prev][Auto▶][Skip→][⌫Clear][▶Loop][Save]  [−] Batch:20 [+] │
└───────────────────────────────────────────────────────────────────────────────────────┘
```

**Batch labeling:** clicking an action labels `batch_size` consecutive frames at once (default 20), then auto-advances to the next batch.

**Controls:**

| Key | Action |
|-----|--------|
| `← / →` | Navigate by batch_size frames |
| `0` – `9` | Label actions 0–9 |
| `a` | Label action 10 (Fwd+Left) |
| `d` | Label action 11 (Fwd+Right) |
| `Space` | Skip to next unlabeled batch |
| `p` | Toggle loop playback of current batch |
| `s` | Save |
| `Del` | Clear labels for current batch |
| `[ / ]` | Decrease / increase batch size |

**Loop playback (`▶ Loop` / `p`):** cycles through the current batch at ~7 fps so you can preview the motion before labeling.

**Saved files** (in the `.bin` folder):

| File | Format | Description |
|------|--------|-------------|
| `actions.npy` | `(N,) int64` | Action per frame, `-1` = unlabeled |
| `actions.csv` | CSV | `frame, x, y, z, action, action_name` |
| `actions_metadata.json` | JSON | Stats, action counts, config |

### `scripts/train.py` — Training

```bash
source venv/bin/activate

# Phase 1: World Model (~2 hours, 100 epochs)
python scripts/train.py \
    --config configs/training/pretrain_world_model.yaml \
    --device cuda

# Phase 2: Decision Transformer (~1 hour, 50 epochs)
python scripts/train.py \
    --config configs/training/train_decision.yaml \
    --device cuda

# Phase 3: Joint Fine-tune (optional)
python scripts/train.py \
    --config configs/training/finetune_joint.yaml \
    --device cuda

# Or run everything at once
./run.sh
```

The Phase 2 trainer auto-loads the Phase 1 world model checkpoint and remaps keys (`tokenizer.X` → `world_model.tokenizer.X`) so weights transfer correctly into the unified model.

### `scripts/realtime_inference.py` — Real-Time Inference UI

Interactive visualization. Loads `.bin` frames, runs the full model, and displays BEV, traversability, decision card, action probabilities, and latency.

```bash
RELLIS_SEQ=/home/rickslab3/Documents/Datasets/RELLIS/dataset/sequences/00000

# Folder mode (most common) — uses default data-dir if omitted
python scripts/realtime_inference.py \
    --mode folder \
    --data-dir $RELLIS_SEQ \
    --checkpoint outputs/decision_train/best_model.pt

# With explicit pose file for ground-truth trajectory overlay
python scripts/realtime_inference.py \
    --mode folder \
    --data-dir $RELLIS_SEQ \
    --checkpoint outputs/decision_train/best_model.pt \
    --pose-file $RELLIS_SEQ/poses.txt

# With explicit world model checkpoint (traversability)
python scripts/realtime_inference.py \
    --mode folder \
    --data-dir $RELLIS_SEQ \
    --checkpoint outputs/decision_train/best_model.pt \
    --world-model-checkpoint outputs/world_model_pretrain/best_model.pt

# No temporal context (single-frame mode)
python scripts/realtime_inference.py --mode folder --no-temporal \
    --data-dir $RELLIS_SEQ

# Simulation (synthetic data, no files needed)
python scripts/realtime_inference.py --mode sim

# Live UDP (Velodyne native protocol)
python scripts/realtime_inference.py --mode udp --port 2368

# Publish decisions via UDP (for external systems)
python scripts/realtime_inference.py --mode folder --publish \
    --publish-host 127.0.0.1 --publish-port 5005
```

**UI panels:**

| Panel | Description |
|-------|-------------|
| BEV Point Cloud (large, left) | Height-colored scatter; ego `▲` at origin; shows filename + frame number |
| Traversability (top-right) | Height-variance proxy — green=flat/safe, yellow=unknown, red=rough/obstacle; colorbar labeled |
| Decision card | Large direction arrow + action name (color-coded by category) + confidence bar + GT match indicator |
| Action Probs | Horizontal bars for all 12 actions, dimmed except winner; percentage labels |
| Info panel | Latency, FPS, vote context, GT match result |

**Folder mode controls:**

| Button | Action |
|--------|--------|
| Load Folder | Open file dialog |
| < Prev / Next > | Navigate frames |
| Play / Pause | Auto-play at target Hz |

**Temporal voting:** uses a rolling buffer of the last N frames; confidence-weighted majority vote across predictions for more stable decisions.

**Traversability:** computed from raw LiDAR geometry (height variance per BEV cell). No world model training needed.

## Encoder Options

| Encoder | Latency | FPS | Notes |
|---------|---------|-----|-------|
| `pointpillars` (default) | ~5 ms | 50+ | Direct BEV projection; recommended |
| `pointnet2` | ~25 ms | ~25 | More expressive; slower |

```bash

python scripts/realtime_inference.py --mode sim --encoder pointnet2
```
